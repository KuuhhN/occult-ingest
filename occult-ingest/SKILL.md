# 神秘学 PDF 入库流程（整合版）

本 Skill 整合了 **OCR 提取** 与 **AI 精读** 两个阶段，将 PDF 转化为结构化笔记。

---

## 触发条件
用户发送 PDF 文件，或说「处理这份资料」「入库」「帮我导读这篇」「格式化原文」等。

---

## 完整流程（5 步）

### 第 1 步：环境检查
确认以下依赖可用（缺了就装）：
- `pdfplumber` — PDF 页面提取
- `tencentcloud-sdk-python-ocr` — 腾讯云 OCR
- `pymupdf` (fitz) — 读取 PDF 页面标签元数据
- 腾讯云 SecretId / SecretKey（向用户索要，用环境变量传递，**绝不写入文件**）

### 第 2 步：OCR 提取原文
```bash
export TENCENT_SECRET_ID="xxx"
export TENCENT_SECRET_KEY="xxx"
python .reasonix/skills/occult-ingest/ocr_extract.py "<pdf路径>" "<书名>" "<作者>"
```

支持 `--vault` 参数指定知识库根目录，或通过 `OCCULT_VAULT_PATH` 环境变量设置。

输出：`02-文献库/经典文献/<书名>_原文.md`
- 格式：`## 第X页` 分页 + 原始 OCR 文本
- 🔒 只写一次，永不覆盖

### 第 3 步：复制 PDF 到 Vault
```bash
cp "<原始PDF路径>" "occult-vault/02-文献库/经典文献/<书名>.pdf"
```

### 第 4 步：读取 PDF 页面标签
```python
import fitz
doc = fitz.open("<pdf路径>")
labels = doc.get_page_labels()
```
获取 Obsidian 中显示的页码方案（如封面 A-H、前言 i-ii、正文 1-247）。
所有笔记中统一使用该标签方案引用页码。

### 第 5 步：AI 精读 + 生成三份笔记

⚠️ **必须精读原文再写笔记，不得跳过阅读直接套模板。** 质量检验：每份笔记必须包含原文中的具体引述或案例作为证据。

#### 5a. 导读（`03-导读/<书名>_导读.md`）
- 📌 一句话总结
- 🎯 核心要点（5-8 条）
- 🗺️ 知识背景（所属体系、难度等级）
- 📖 阅读导航（建议顺序 + 表格）
- 💡 关键术语表
- 🔗 延伸阅读
- 📕 PDF 链接
- **末尾必须加导航链接**：→ [[摘要目录]] → [[精读笔记]]

#### 5b. 摘要目录（`05-摘要/<书名>_摘要目录.md`）
- 按章→节分层
- 每节：页码范围 + 一段话概括（3-5 句）
- 📕 PDF 链接
- **末尾必须加导航链接**：→ [[精读笔记]] → [[.pdf|PDF]]

#### 5c. 精读笔记（`06-笔记/<书名>_精读笔记.md`）
- 保留原文 ~50% 信息量
- **每个二级标题必须标注页码范围**
- 📕 PDF 链接

### 第 6 步：更新知识条目
在 `01-知识库/<体系>/` 下创建或更新对应条目，链接导读、摘要目录、精读笔记、原文、PDF。

### 第 7 步：记录日志
在 `04-日志/` 创建或更新当日日志（格式 `YYYY-MM-DD.md`）。每次入库追加到当日日志中，同一天多份资料汇总在同一文件。

---

## 页码规范
- **所有页码使用 Obsidian PDF 标签**（从第 4 步读取）
- 格式示例：`（i–ii）`、`（1–60）`
- 不混用 PDF 物理页码或印刷页码

## 文件命名规范
```
02-文献库/经典文献/<书名>_原文.md
02-文献库/经典文献/<书名>.pdf
03-导读/<书名>_导读.md
05-摘要/<书名>_摘要目录.md
06-笔记/<书名>_精读笔记.md
01-知识库/<体系>/<书名>.md
```
### 文件夹结构
```
00-MOC/      总览索引
01-知识库/   知识条目（按体系分子目录）
02-文献库/   原文 PDF + OCR 原文
03-导读/     快速入门导读
04-日志/     每日学习记录
05-摘要/     分层摘要目录
06-笔记/     精读笔记（~50% 密度）
99-模板/     笔记模板
```
### 导航链规则
- 导读末尾 → [[摘要目录]] → [[精读笔记]]
- 摘要末尾 → [[精读笔记]] → [[.pdf|PDF]]

## 开源 / 通用化说明

本工具已做通用化处理：
- **路径**：通过 `--vault` 参数或 `OCCULT_VAULT_PATH` 环境变量配置知识库根目录
- **书名/作者/出版社**：全部作为 CLI 参数传入，无硬编码
- **印刷页眉**：通过 `--headers` 参数传入，适应不同书籍（已废弃，精排版步骤已移除）
- **API 密钥**：通过环境变量传递，**绝不写入文件**
